# Team-division（チーム分けアプリ）

メンバーを登録して、指定した人数（または指定したチーム数）でランダムにチーム分けできるWebアプリです。

## 特徴

- **ビルド不要** — `index.html` をブラウザで開くだけで動作します
- **メンバー登録** — 名前を1人ずつ、または改行・カンマ・スペース区切りでまとめて追加できます
- **自動保存** — 登録したメンバーはブラウザの localStorage に保存され、次回も残ります
- **保存リストで使い回し** — メンバーに名前を付けて複数のリストとして保存し、いつでも呼び出して使い回せます
- **2つの分け方**
  - 1チームの人数で指定（例: 4人ずつ）
  - チーム数で指定（例: 3チームに分ける）
- **役割のランダム割り当て** — 書記・発表・司会・タイムキーパーなどの役割を各チームにランダムで割り当て。役割は自由に追加・削除でき、人数が役割より少ないチームでは1人が複数役割を兼任します
- **均等配分** — 割り切れない場合も人数差が最大1人になるように振り分けます
- **再シャッフル** — 同じ設定でワンクリックで分け直せます
- **1人ずつ発表（ルーレット）** — チーム分け後、1人ずつルーレットで所属チームを発表します。結果は音声でも読み上げ（ブラウザの音声合成、ON/OFF切替可）、次の人の番になると前の人の結果は自動で隠れます
- **チームメモ（5W1H）** — チームごとに番号が振られ、いつ／どこで／だれが／なにを／なぜ／どうやって＋備考を枠線付きで記録できます。内容は端末の localStorage に自動保存。チーム番号を入力して目的のメモへすぐ移動できます
- **CSV書き出し** — 各チームのメモを CSV で出力。別システム（AI）での要約・整理を想定し、**1行＝1項目**の long 形式（`team_number, team_name, members, element_key, element_label, content`）で、UTF-8 BOM 付き（Excelでも文字化けしません）

## メモのCSV構成（AI連携を想定）

CSV は LLM が解釈しやすいよう、1行に1つの 5W1H 項目を持つ正規化（long）形式です。

| 列 | 内容 | 例 |
| --- | --- | --- |
| `team_number` | チーム番号 | `1` |
| `team_name` | チーム名 | `チーム1` |
| `members` | メンバー（` / ` 区切り） | `山田 / 田中 / 佐藤` |
| `element_key` | 5W1Hの英語キー | `when` / `where` / `who` / `what` / `why` / `how` / `notes` |
| `element_label` | 画面表示ラベル | `いつ (When)` |
| `content` | 入力された内容 | `来週月曜の朝礼で` |

別端末との完全なリアルタイム同期は行いません（バックエンド不要の静的サイトのため）。メモは各端末の localStorage に保存され、CSV で受け渡し・集約できます。

## 公開 URL（GitHub Pages）

このサイトは **「ブランチ配信」** で公開します（ビルド不要・ワークフロー不要。`main` ブランチ直下の静的ファイルをそのまま配信）。

- 公開 URL: <https://capinfo0000.github.io/team-division/>

### 初回だけ必要な設定（リポジトリ管理者）

GitHub の **Settings → Pages → Build and deployment** で次を設定して保存してください。

- **Source**: `Deploy from a branch`
- **Branch**: `main` / `(root)`

保存すると GitHub が自動でビルド（`pages build and deployment`）して公開します。以降は `main` に push するたび自動で再公開されます。`.nojekyll` を置いているため、静的ファイルはそのまま配信されます。

## 使い方

1. ブラウザで上記の公開 URL を開く（またはローカルの `index.html` を開く）
2. メンバーの名前を入力して「追加」
3. 「1チームの人数」か「チーム数」を選んで数値を入力
4. 「チーム分けする」を押すと結果が表示されます
5. 必要に応じて「1人ずつ発表（ルーレット）」で1人ずつ発表
6. 各チームの話し合いを「チームメモ（5W1H）」に記録し、「CSVで書き出す」で出力
