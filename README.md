# Team-division（チーム分けアプリ）

メンバーを登録して、指定した人数（または指定したチーム数）でランダムにチーム分けできるWebアプリです。

## 2つのモード（上部タブで切替）

- **🎰 ルーレット抽選** — 人数とチーム数（と役割）を設定してスタート。1人ずつルーレット演出でチーム・役割を発表し、「次の人へ」で順番に進みます。最後に結果一覧を表示
- **📝 リストで分ける** — 名前を登録して、人数/チーム数を指定して一括チーム分け

## 特徴

- **ビルド不要** — `index.html` をブラウザで開くだけで動作します
- **メンバー登録** — 名前を1人ずつ、または改行・カンマ・スペース区切りでまとめて追加できます
- **自動保存** — 登録したメンバーはブラウザの localStorage に保存され、次回も残ります
- **保存リストで使い回し** — メンバーに名前を付けて複数のリストとして保存し、いつでも呼び出して使い回せます
- **2つの分け方**
  - 1チームの人数で指定（例: 4人ずつ）
  - チーム数で指定（例: 3チームに分ける）
- **役割のランダム割り当て** — 書記・発表・司会・タイムキーパーなどの役割を各チームにランダムで割り当て。役割は自由に追加・削除でき、人数が役割より少ないチームでは1人が複数役割を兼任します
- **均等配分** — 割り切れない場合も人数差が最大1人になるように振り分けます
- **再シャッフル** — 同じ設定でワンクリックで分け直せます

## 使い方

1. `index.html` をブラウザで開く
2. メンバーの名前を入力して「追加」
3. 「1チームの人数」か「チーム数」を選んで数値を入力
4. 「チーム分けする」を押すと結果が表示されます

## 会議メモ機能（共有付箋ボード）— Xserver(PHP+MySQL)用

チーム分け後に「共有メモ」を作成すると、チームごとに**番号**が発行されます。
参加者は別端末でその番号を入力すると、同じ付箋ボードを開いてメモを書き込めます。
メモはMySQLに保存され、CSV出力できます（カテゴリ別なので意見の要約に向いた構成）。

### セットアップ（Xserver）
1. Xserverサーバーパネル →「MySQL設定」でデータベースとユーザーを作成
2. phpMyAdmin で対象DBを選び、`api/schema.sql` の内容を実行（テーブル作成）
3. `api/config.php` をDB名・ユーザー・パスワードに書き換え
4. `index.html` `style.css` `app.js` と `api/` フォルダ一式を同じディレクトリにアップロード
   （例: `/public_html/projects/teamdivision/`）
5. ブラウザで `index.html` を開く → チーム分け→「🔗 共有メモを作成」

> メモ機能はPHP+MySQLが必要なため、GitHub Pages（静的のみ）では動きません。
> チーム分け・ルーレット・音声読み上げは静的環境でも動作します。

### CSVの構成（要約しやすい形）
`ミーティングID, チーム, カテゴリ, メモ, 投稿者, 日時` の1メモ=1行。
カテゴリ（議題/良い点/課題/アイデア/質問/メモ）で集計すれば「どんな意見が多いか」を要約しやすくしています。
