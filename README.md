# Team-division（チーム分けアプリ）

メンバーを登録して、指定した人数（または指定したチーム数）でランダムにチーム分けできるWebアプリです。

## 2つのモード（上部タブで切替）

- **🎰 ルーレット抽選** — 人数とチーム数（と役割）を設定してスタート。「発表！」を押すと1人ずつルーレット演出でチーム・役割を発表し、「次の人へ」で順番に進みます。発表前は「？」表示なので**前の人の結果は次の人に見えません**。最後に結果一覧を表示
- **📝 リストで分ける** — 名前を登録して、人数/チーム数を指定して一括チーム分け

## 特徴

- **ビルド不要** — `index.html` をブラウザで開くだけで動作します
- **メンバー登録** — 名前を1人ずつ、または改行・カンマ・スペース区切りでまとめて追加できます
- **自動保存** — 登録したメンバーはブラウザの localStorage に保存され、次回も残ります
- **保存リストで使い回し** — メンバーに名前を付けて複数のリストとして保存し、いつでも呼び出して使い回せます
- **2つの分け方**
  - 1チームの人数で指定（例: 4人ずつ）
  - チーム数で指定（例: 3チームに分ける）
- **役割のランダム割り当て** — 書記・発表・司会・タイムキーパーなどの役割を各チームにランダムで割り当て。役割は自由に追加・削除でき、人数が役割より少ないチームでは1人が複数役割を兼任します
- **均等配分** — 割り切れない場合も人数差が最大1人になるように振り分けます
- **再シャッフル** — 同じ設定でワンクリックで分け直せます
- **音声読み上げ** — ルーレットの発表結果をブラウザの音声合成で読み上げ（画面の「🔊 結果を音声で読み上げる」でON/OFF）
- **チームメモ（5W1H）** — 結果一覧の下に、チームごとに番号付きのメモ欄を表示。**1チーム＝1つの自由記述欄**で、薄い文字（プレースホルダー）が 5W1H の記入例として表示されます
- **別端末でメモ共有（任意・サーバー連携）** — 「📡 チーム番号を発行」でチームごとに番号を発行。別端末の「🗒️ メモを開く」タブにその番号を入力すると、同じメモを編集できます（PHP + MySQL のバックエンド設定が必要）。未設定時は各端末のローカル保存のみ
- **CSV書き出し** — 各チームのメモを CSV で出力。別システム（AI）での要約・整理を想定し、**1チーム＝1行**（`team_number, team_name, members, memo`）＋ UTF-8 BOM 付き。サーバー連携時は**全端末で書かれた最新メモを集約**して出力します

## メモのCSV構成（AI連携を想定）

CSV は LLM が解釈しやすいよう、1チームを1行で表します。

| 列 | 内容 | 例 |
| --- | --- | --- |
| `team_number` | チーム番号 | `1` |
| `team_name` | チーム名 | `Aチーム` |
| `members` | メンバー（` / ` 区切り） | `山田 / 田中 / 佐藤` |
| `memo` | メモ本文（5W1Hを意識した自由記述。改行を含む場合あり） | `いつ：来週月曜… / どこで：第1会議室…` |
| `team_code` | 発行された番号（サーバー連携時のみ） | `359582` |

## 使い方

1. `index.html` をブラウザで開く（または公開URLを開く）
2. **🎰 ルーレット抽選**: 人数・チーム数・役割を設定して「スタート」→「発表！」で1人ずつ発表
3. **📝 リストで分ける**: 名前を登録 →「ルーレットでスタート」でチームごとに発表
4. 結果一覧の下の「チームメモ（5W1H）」に話し合いを記録（サーバー連携時は「📡 チーム番号を発行」→各チームに番号を配布→別端末の「🗒️ メモを開く」で入力）
5. 「⬇ CSVで書き出す」で各チームのメモを出力

## 別端末でメモを共有する（PHP + MySQL / Xserver 想定）

ブラウザは MySQL に直接つながらないため、間に PHP の API を置きます：`ブラウザ → api/memo.php → MySQL`。

セットアップ手順（Xserver スタンダードの例）:

1. **DB作成**: サーバーパネル → 「MySQL設定」でデータベースとユーザーを作成（権限付与）
2. **テーブル作成**: phpMyAdmin で `db/schema.sql` を実行
3. **設定ファイル**: `api/config.sample.php` を `api/config.php` にコピーし、DBホスト/DB名/ユーザー/パスワードを記入（`config.php` は公開禁止・Git管理外）
4. **アップロード**: `api/` フォルダ（`memo.php` / `config.php` / `.htaccess`）を公開領域（例: `public_html/team-division/api/`）に設置
5. **画面側の設定**: `app.js` の `var API_BASE = "";` に API のURLを設定
   - 例: `var API_BASE = "https://あなたのドメイン/team-division/api/memo.php";`
   - 画面を GitHub Pages、API を Xserver と別ドメインにする場合は、`config.php` の `cors_origin` に `https://capinfo0000.github.io` を設定（同一ドメインに置くなら不要）

### API 仕様（参考）

| メソッド | パス | 用途 |
| --- | --- | --- |
| `POST` | `?action=create`（body: `{teams:[{team_number,team_name,members}]}`） | チームを登録し、番号（`team_code`）と集約コード（`session_code`）を発行 |
| `GET` | `?team=コード` | そのチームのメモを取得 |
| `POST` | `?action=save`（body: `{team_code, memo}`） | メモを保存 |
| `GET` | `?session=コード` | 同じ回の全チームを取得（CSV集約用） |
